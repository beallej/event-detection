How To Use JWS (Java WordNet::Similarity) - beta

David Hope, 2008, University Of Sussex



What You Will Need


  • A copy of WordNet (WordNet 1.6 ... 3.0) http://wordnet.princeton.edu/obtain

  • Information Content Files http://wn-similarity.sourceforge.net/

  • edu.sussex.nlp.jws.beta.xx.jar included in this download (JWS)

  • edu.mit.jwi_2.1.4.jar included in this download (JWI)


Where You Put All This Stuff: An Example


Say that you want to use WordNet 3.0 (the same procedure applies to WordNet 1.6...2.1)


1. Get a copy of WordNet 3.0 and place it in a folder (you only need the dict folder)


2. Get the corresponding version of the Information Content files and place them in the same folder that WordNet 3.0 is in.






3. Make sure that you have a pointer to JWS + JWI. For example, add the path to the JWS + JWI jar files to your CLASSPATH



Note that you can install every version of WordNet 1.6 ... 3.0. You create a folder for each version of WordNet and put the corresponding Information Content files within that folder. You could then use any version of WordNet 1.6 ... 3.0 using JWS.



If you wanted to do this for the Perl version you would have to get the specific code for each version of WordNet and install it.


Important


WordNet 3.0 has an infinite loop (inhibit > < restrain). I have applied the patch that Princeton provide (see link below) directly to the WordNet 3.0 verb data file. Similarity measures involving “inhibit” and/or “restrain” may have slightly different results than those of the Perl version. We use Princeton's patch as that seems to be the 'safer' option (no more infinite loops!); I am not sure exactly how the Perl version gets round this problem.


http://wordnet.princeton.edu/~ben/WN3.loop.patch
