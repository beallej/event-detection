AdaptedLesk - beta

David Hope, 2008, University Of Sussex


Gets the similarity between any combination of


( word1, word1sense#, word2, word2sense#, POS )


Use:


lesk.lesk( cat, 1, dog, 2, n )

lesk.lesk( cat, dog, 2, n )

lesk.lesk( cat, 1, dog, n )

lesk.lesk( cat, dog, n )

lesk.max( cat, dog, n )



Brief Description:


Follows Banerjee & Pedersen's description


-- gloss overlap lengths are squared

e.g.

“cat” = 1

“cat drinks” = 4

“cat drinks milk” = 9


-- a gloss overlap must not consist of all function words

e.g.

“of the” = 0

“cream of the crop” = 16

-- we get the sum of the gloss overlap lengths squared for all relations in WordNet for word 1, sense X, word 2, sense Y


Notes:


We use the same scoring mechanism as described in Banerjee & Pedersen. However, here, we use all the related glosses i.e. all the data available to us from WordNet, not just those relations listed in the (lesk-)relations.dat file that comes with the Perl version. Thus, we will get different (better?) scores to those that the Perl version returns.


The measure is slow compared to, say, Jiang & Conrath as we must match, with some care, against all possible overlaps in glosses without matching subsets of the longest overlaps that we have found already. We must also check for overlaps that contain all function words if we are using the stop list and/or the lemmatiser. This is a computationally expensive process.


Refs:


Banerjee S. and Pedersen T. 2002. An Adapted Lesk Algorithm for Word Sense Disambiguation Using WordNet. In Proceeding of the Fourth International Conference on Computational Linguistics and Intelligent Text Processing (CICLING-02). Mexico City.
